# Technische Beschreibung

Erstelle eine vollständige funktionale Beschreibung des Projekts **mit allen technischen Details**.

## Anweisungen

1. Beschreibe den Zweck der Anwendung
2. Liste alle Features auf (Backend und Frontend getrennt)
3. Erkläre die technische Implementierung
4. Beschreibe die Architektur und verwendeten Technologien
5. Dokumentiere APIs und Datenflüsse

## Optionale Ausgabe in Datei

Falls der Benutzer `> dateiname` angibt:
- Schreibe die Beschreibung als Markdown in diese Datei
- Bei Sprachsuffix (z.B. `.de.md` oder `.en.md`): Schreibe in der entsprechenden Sprache

## Format

Inkludiere technische Details wie:
- Verwendete Frameworks und Libraries
- API Endpoints und deren Struktur
- Datenbank-Schema
- State Management Patterns
- Build und Deployment Prozesse
