# Funktionale Beschreibung

Erstelle eine vollständige funktionale Beschreibung des Projekts **ohne technische Details**.

## Anweisungen

1. Beschreibe den Zweck der Anwendung
2. Liste alle Features auf (Backend und Frontend getrennt)
3. Erkläre die Benutzerinteraktionen
4. Beschreibe die unterstützten Plattformen

## Optionale Ausgabe in Datei

Falls der Benutzer `> dateiname` angibt:
- Schreibe die Beschreibung als Markdown in diese Datei
- Bei Sprachsuffix (z.B. `.de.md` oder `.en.md`): Schreibe in der entsprechenden Sprache

## Format

Fokus auf **Was** die Anwendung tut, nicht **Wie** sie es technisch umsetzt.

Beispiel:
- "Benutzer können Konfigurationen erstellen und verwalten"
- NICHT: "React Context speichert den State in einem Provider"
